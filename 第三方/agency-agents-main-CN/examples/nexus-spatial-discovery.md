## 1. 机会

### 如何发现

跨多个来源的网络研究识别出三个趋同趋势：

- **AI基础设施/编排**是增长最快的软件类别（AI编排市场2026年估值约135亿美元，22%+复合年增长率）
- **空间计算**（Vision Pro、WebXR）正在成熟，但缺乏杀手级企业应用
- 每个现有的AI工作流工具（LangSmith、n8n、Flowise、CrewAI）都是**扁平的2D仪表板**

### 概念：Nexus Spatial

空间计算中的AI智能体指挥中心——一个VisionOS + WebXR应用，提供沉浸式3D指挥中心，用于编排、监控和与AI智能体交互。用户将智能体流水线可视化为3D节点图，在空间面板中监控实时输出，在3D空间中拖放构建工作流，并在共享空间环境中协作。

### 为什么此机构具有独特优势

该机构拥有深厚的空间计算专业知识（XR开发者、VisionOS工程师、Metal专家、界面架构师），以及完整的工程、设计、营销和运营栈——对于一个既需要空间计算精通又需要企业软件严谨性的产品来说，这是一种罕见的组合。

- 装机量：全球约100万台（销量较发布时下降95%）

- **含义：** 不要以VisionOS为先导。以Web为先导，添加WebXR，原生VisionOS最后。

## 3. 技术架构

**智能体：** 后端架构师

### 系统概览

8服务架构，具有清晰的 ownership 边界，专为水平扩展和供应商无关的AI集成而设计。

```
+------------------------------------------------------------------+
|                     客户端层                                       |
|  VisionOS原生 (Swift/RealityKit)  |  WebXR (React Three Fiber)    |
+------------------------------------------------------------------+
                              |
+-----------------------------v------------------------------------+
|                      API网关 (Kong / AWS API GW)                  |
|  速率限制 | JWT验证 | WebSocket升级 | TLS                        |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      服务层                                       |
|  Auth | Workspace | Workflow | Orchestration (Rust) |             |
|  Collaboration (Yjs CRDT) | Streaming (WS) | Plugin | Billing    |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                      数据层                                       |
|  PostgreSQL 16 | Redis 7集群 | S3 | ClickHouse | NATS            |
+------------------------------------------------------------------+
                              |
+------------------------------------------------------------------+
|                    AI供应商层                                      |
|  OpenAI | Anthropic | Google | 本地模型 | 自定义插件               |
+------------------------------------------------------------------+
```

### 技术栈

| 组件 | 技术 | 理由 |
|------|------|------|
| 编排引擎 | **Rust** | 亚毫秒调度、零GC暂停、智能体沙箱的内存安全 |
| API服务 | TypeScript / NestJS | CRUD密集型服务的开发者效率 |
| VisionOS客户端 | Swift 6, SwiftUI, RealityKit | 一流的空间计算与Liquid Glass |
| WebXR客户端 | TypeScript, React Three Fiber | 生产级WebXR与React组件模型 |
| 消息代理 | NATS JetStream | 轻量级、精确一次交付、比Kafka更简单 |
| 协作 | Yjs (CRDT) + WebRTC | 无冲突的并发3D图编辑 |
| 主数据库 | PostgreSQL 16 | JSONB用于灵活配置、行级安全用于租户隔离 |

### 核心数据模型

14张表覆盖：
- **身份与访问：** users、workspaces、team_memberships、api_keys
- **工作流：** workflows、workflow_versions、nodes、edges
- **执行：** executions、execution_steps、step_output_chunks
- **协作：** collaboration_sessions、session_participants
- **凭证：** provider_credentials（AES-256-GCM加密）
- **计费：** subscriptions、usage_records
- **审计：** audit_log（仅追加）

### 节点类型注册表

```
内置节点类型：
  ai_agent          -- 使用提示调用AI供应商
  prompt_template   -- 使用变量渲染模板
  conditional       -- 基于表达式路由
  transform         -- 沙箱代码片段（JS/Python）
  input / output    -- 工作流入口/出口点
  human_review      -- 暂停等待人工审批
  loop              -- 重复子图
  parallel_split    -- 扇出到分支
  parallel_join     -- 等待分支
  webhook_trigger   -- 外部HTTP触发器
  delay             -- 定时暂停
```

### WebSocket通道

通过WSS进行实时流式传输，具有：
- 每通道序列号用于排序
- 间隙检测与重放请求
- 落后超过1000个事件时的快照恢复
- 低功耗设备的客户端节流

### 安全架构

| 层 | 机制 |
|---|------|
| 用户认证 | OAuth 2.0（GitHub、Google、Apple）+ 邮箱/密码 + 可选TOTP MFA |
| API密钥 | SHA-256哈希、范围限定、可选过期 |
| 服务间通信 | 通过服务网格的mTLS |
| WebSocket认证 | 30秒过期的一次性票据 |
| 凭证存储 | 信封加密（AES-256-GCM + AWS KMS） |
| 代码沙箱 | gVisor/Firecracker微VM（无网络、256MB RAM、30秒CPU） |
| 租户隔离 | PostgreSQL行级安全 + S3 IAM策略 + NATS主题范围限定 |

## 4. 品牌策略

**智能体：** 品牌守护者

### 定位

**品类创造优于品类竞争。** Nexus Spatial定义了一个新类别——**空间AI运维（SpatialAIOps）**——而不是在拥挤的AI可观测性仪表板空间中争夺位置。

**定位声明：** 对于管理复杂AI智能体工作流的技术团队，Nexus Spatial是沉浸式3D指挥中心，提供智能体编排的空间感知，与扁平2D仪表板不同，因为空间计算将监控从阅读仪表板转变为驻留在你的基础设施中。

### 名称验证

"Nexus Spatial"**验证为强有力：**
- "Nexus"连接到NEXUS编排框架（Network of EXperts, Unified in Strategy）
- "Nexus"独立含义为"中心连接点"——完美适合指挥中心
- "Spatial"是苹果和行业已标准化的行业描述符
- 语音平衡：三个音节，然后两个
- **需要行动：** 在尼斯分类第9、42和38类中进行商标清查

### 关键发现：调试是杀手级用例

运行时追踪在工作流结构上的空间叠加解决了一个真实的、量化的痛点，没有2D工具能处理好。这个工作流应该获得最多的设计和工程投入。

### 关键设计洞察

空间为**结构性**任务（放置、连接、重排节点）增加价值，但为**参数性**任务（文本输入、配置）制造摩擦。界面必须无缝融合空间和2D模式——锚定在空间位置的2D面板。

### 7条设计原则

1. **空间必须证明其价值** —— 如果2D更清晰，就用2D。每次审查都应问："这用扁平的会更好吗？"
2. **可瞥见先于可检查** —— 关键信息在2秒内通过颜色、大小、运动、位置感知
3. **免手操作是基线** —— 注视 + 语音覆盖所有读取/导航操作；双手增加精度但不是必需的
4. **尊重认知引力** —— 扩展2D心智模型（从左到右的流程），不要替换它们；z轴增加分层
5. **渐进式空间复杂度** —— 新用户从接近2D开始；空间能力随信心增长而展现
6. **物理隐喻，数字能力** —— 节点被"拾起"（物理的），但也可以复制和版本化（数字的）
7. **安静是功能** —— 健康的系统感觉平静；颜色和运动标示偏离正常

## 9. 空间界面架构

**智能体：** XR界面架构师

### 指挥剧场

工作区围绕用户组织为弧形剧场：

```
                        概览天篷
                     （流水线拓扑）
                    ~~~~~~~~~~~~~~~~~~~~~~~~
                   /                        \
                  /     焦点弧（120度）       \
                 /    主要节点图工作区         \
                /________________________________\
               |                                  |
    左侧       |        用户位置                  |       右侧
    工具       |        （原点 0,0,0）            |       工具
    轨道       |                                  |       轨道
               |__________________________________|
                \                                /
                 \      架子（视线以下）         /
                  \   智能体状态、快捷工具      /
                   \_________________________ /
```

- **焦点弧**（120度，1.2-2.0米）：主要节点图工作区
- **概览天篷**（上方，2.5-4.0米）：迷你流水线拓扑 + 健康热图
- **工具轨道**（左右侧翼）：智能体库、监控、日志
- **架子**（视线以下，0.8-1.0米）：运行/停止、撤销/重做、快捷工具

### 三层深度系统

| 层 | 深度 | 内容 | 不透明度 |
|---|------|------|----------|
| 前景 | 0.8 - 1.2米 | 活动面板、检查器、模态框 | 100% |
| 中景 | 1.2 - 2.5米 | 节点图、连接、工作区 | 100% |
| 背景 | 2.5 - 5.0米 | 概览地图、环境状态 | 40-70% |

### 3D中的节点图

**数据流向用户。** 节点按执行顺序沿z轴排列：

```
用户（此处）
  z=0.0m   [输出节点]     -- 结果
  z=0.3m   [变换节点]     -- 处理器
  z=0.6m   [智能体节点]   -- LLM调用
  z=0.9m   [检索节点]     -- RAG、API
  z=1.2m   [输入节点]     -- 触发器
```

并行分支水平展开（x轴）。条件分支垂直展开（y轴）。

**节点表示（3个LOD）：**
- **LOD-0**（静止，>1.5米）：12x8厘米磨砂玻璃矩形，带类型图标、名称、状态发光
- **LOD-1**（悬停，400毫秒注视）：扩展到14x10厘米，显示端口、上次运行信息
- **LOD-2**（选中）：滑到前景，扩展到30x40厘米详情面板，支持实时配置编辑

**连接作为发光管道：**
- 静止时4毫米直径，传输数据时8毫米
- 按数据类型颜色编码（白色=文本、青色=结构化、品红色=图像、琥珀色=音频、绿色=工具调用）
- 动画粒子显示流向和速度
- 同层之间超过3条并行时自动捆绑

### 7种智能体状态

| 状态 | 边缘发光 | 内部 | 声音 | 粒子 |
|------|---------|------|------|------|
| 空闲 | 稳定绿色，低 | 静态磨砂玻璃 | 无 | 无 |
| 排队 | 脉冲琥珀色，1Hz | 微弱旋转 | 无 | 输入端缓慢漂移 |
| 运行中 | 稳定蓝色，中等 | 动画闪烁 | 柔和空间嗡鸣 | 连接上快速流动 |
| 流式传输 | 蓝色 + 输出流 | 闪烁 + 文本片段 | 嗡鸣 | 文本片段向前流动 |
| 完成 | 闪白，然后绿色 | 静态 | 完成提示音 | 无 |
| 错误 | 脉冲红色，2Hz | 红色色调 | 警报音（一次） | 无 |
| 暂停 | 稳定琥珀色 | 冻结帧 + 暂停图标 | 无 | 原地冻结 |

| 区域切换 | 500毫秒 | 当前区域横向滑出，新区域滑入 |
| 窗口到沉浸 | 1000毫秒 | 边框溶解，节点扩展到完整空间位置 |

### 舒适度措施

- 没有用户操作不启动相机移动
- 稳定地平线（水平面永不倾斜）
- 主要交互在0.8-2.5米内，视线正负15度
- 45分钟后休息提示（环境灯光变化，非模态）
| 快速移动时的外围渐晕
- 所有常用控件可在手臂自然下垂时访问（仅手腕/手指）

## 10. 跨智能体综合

### 所有8个智能体的一致点

1. **2D优先，空间其次。** 每个智能体独立得出这个结论。先构建一个优秀的Web仪表板，然后渐进地添加空间能力。

2. **调试是杀手级用例。** 产品趋势研究员、UX研究员和XR界面架构师都汇聚于此：运行时追踪在工作流结构上的空间叠加是3D真正胜过2D的地方。

3. **WebXR优于VisionOS用于初始覆盖。** Vision Pro约100万的装机量无法支撑一个业务。浏览器中的WebXR是分发解锁。

4. **"作战室"协作场景。** 多个智能体强调协作事件响应是最强的空间价值主张——团队进入共享3D空间一起调试故障流水线。

5. **渐进式揭示至关重要。** UX研究、空间UI和支持都强调空间复杂度必须逐渐展现，永远不要一次性呈现给首次用户。

6. **语音是高级用户的加速器。** UX研究员和XR界面架构师都将语音命令识别为"空间计算的命令行"——对无障碍和专家效率至关重要。

### 需要解决的关键张力

| 张力 | A方立场 | B方立场 | 需要的解决方案 |

| **定价** | 增长黑客：$29-59/用户/月 | 趋势研究员：$99-249/用户/月 | Beta中A/B测试 |

### 此练习展示了什么

这份发现文档由8个专业智能体并行运行产出，每个都为共同目标带来深厚的领域专业知识。智能体独立得出一致的结论，同时揭示了任何单一通才难以产出的领域特定洞察：

- **产品趋势研究员**发现了令人清醒的Vision Pro销售数据，重新定义了整个策略
- **后端架构师**设计了Rust编排引擎，这是任何以营销为焦点的团队都不会考虑的
- **品牌守护者**创建了一个品类（"SpatialAIOps"），而不是在现有品类中竞争
- **UX研究员**发现空间计算为参数任务制造摩擦——一个反直觉的发现
- **XR界面架构师**设计了"数据流向你"的拓扑，映射到自然空间认知
- **项目管理者**识别了三个可能破坏整个时间线的关键瓶颈角色
- **增长黑客**设计了特定于空间计算天生可分享性的病毒循环
- **支持响应者**将产品自身的AI能力转化为支持差异化优势

结果是一份全面的、跨职能的产品计划，可以作为实际开发的基础——由一个AI智能体机构在单次会话中协作产出。
