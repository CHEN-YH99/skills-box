# Synex Cinematic Scroll Skill

用于实现高端滚动叙事页面的专用技能。调用本技能时，不要把任务理解成“加一点滚动动画就完事”，而是要按分镜、分层、转场、节奏、素材规范一起设计和落地。

本技能已整合：

- 现有参考视频反推出来的分镜与动画 DNA
- [`需求文档.md`](../../../../需求文档.md) 中的“通用分层视觉素材规范”
- 面向 `GSAP + ScrollTrigger` 的工程实现约束

## 1. 什么时候必须用这个技能

满足以下任一条件时使用：

- 用户明确提到 `synex-cinematic-scroll`
- 需求是整屏滚动官网、品牌故事页、产品发布页、营销首页
- 需求包含固定导航、分镜切换、前中后景视差、文字遮罩入场、白场/模糊转场
- 设计目标是“高级、电影感、叙事感、空间感”，而不是普通卡片列表页面

如果页面已经有成熟设计系统，先保留原有视觉语言，只把本技能的“分层与动画方法”融进去，不要强行换皮。

## 2. 核心目标

你要实现的不是单个动画，而是一整套可持续复用的滚动叙事系统：

1. 每个 `section` 是一个独立分镜
2. 每个分镜必须有单一主视觉锚点
3. 每个分镜至少包含 `bg / mid / fg` 三层
4. 每个分镜都要具备清晰的入场、停驻、离场逻辑
5. 所有分镜之间通过统一节奏和统一转场语言衔接

## 3. 分层视觉素材规范

这是硬规范，不是建议。素材不达标，就不要假装自己能做出真正的空间叙事效果。

### 3.1 最低交付要求

每个 section 至少准备 3 层：

- `bg` 背景层：不透明 `WebP`，建议 `3840x2160`
- `mid` 主体层：透明 `PNG/WebP`，建议长边 `>= 2500`
- `fg` 前景层：透明 `PNG/WebP`，建议长边 `>= 2500`

### 3.2 每层职责

| 层级 | 内容职责 | 动画职责 |
|------|----------|----------|
| `bg` | 环境、空间、天空、远景氛围 | 慢速大位移、轻微缩放、模糊离场 |
| `mid` | 主体建筑、核心装置、主卡片、主角色 | 主叙事锚点、主体入场、中速视差 |
| `fg` | 岩石、玻璃、雾层、枝叶、边缘遮挡物 | 遮挡、穿越感、镜头擦过、叙事强化 |

### 3.3 素材构图要求

- 三层素材必须共享同一透视关系、主光方向、地平线
- `bg` 不能过度抢戏，避免信息密度超过 `mid`
- `mid` 周围必须预留 8% 到 12% 安全出血
- `fg` 必须有真实遮挡价值，不能只是装饰碎片
- 文案、按钮、导航禁止烘焙进位图
- 透明边缘必须干净，不能带脏灰边和半透明残影

### 3.4 目录与命名规范

统一按 section 管理：

```text
apps/frontend/public/images/sections/
  s01-hero/
    s01-hero__bg.webp
    s01-hero__mid-main.webp
    s01-hero__fg-rocks.webp
  s02-features/
    s02-features__bg.webp
    s02-features__mid-card-stack.webp
    s02-features__fg-glass.webp
```

命名规则：

- section 目录：`sXX-scene-name`
- 文件命名：`sectionKey__layer-variant.ext`
- `layer` 只允许 `bg` / `mid` / `fg`

### 3.5 不达标时的处理规则

如果某个 section 没有满足三层：

- 不要硬做完整 3D 叙事动画
- 明确告诉用户素材不足会导致效果降级
- 可临时降级为轻视差 + 文案动画
- 若任务目标明确要求“电影级空间感”，优先推动补素材，而不是拿代码硬凹

## 4. 动画 DNA

这是所有场景都要尽量遵守的底层规律：

1. 永远多平面：背景层、内容层、浮层至少三层，避免单层平移
2. 转场先“去清晰”，再“进清晰”：`blur + opacity + scale` 比单纯 fade 更高级
3. 一镜只强调一个主对象：每段只保留一个主视觉锚点
4. 导航不参与戏剧化：导航固定、稳定、低运动
5. 整体节奏遵循“快入慢停”：进入快、停驻稳、离场干净

## 5. 分镜基线

参考视频中可复用的场景节奏如下：

1. `0s-6s` Hero 首屏
2. `6s-11s` 三列能力卡
3. `11s-17s` 中央焦点卡 + 曲线背景
4. `17s-24s` 节点网络图
5. `24s-27s` Integrations 生态环
6. `27s-31s` 暗色对比表
7. `31s-37s` CTA + 底部信息

后续做新项目时，不要求死抄这 7 段内容，但要尽量遵守这种节奏结构：

- Hero 建立世界观
- Features 解释能力
- Focus 展示核心卖点
- Network / Integrations 展示体系连接
- Compare 建立价值对比
- CTA 收束转化

## 6. 场景实现规范

### 6.1 容器结构

每个分镜建议至少包含：

```html
<section class="scene scene-hero">
  <div class="scene__bg"></div>
  <div class="scene__mid"></div>
  <div class="scene__fg"></div>
  <div class="scene__content"></div>
</section>
```

推荐职责：

- `scene__bg`：背景位图承载层
- `scene__mid`：主体位图承载层
- `scene__fg`：前景遮挡层
- `scene__content`：标题、正文、按钮、标签、数据 UI

### 6.2 导航规则

- 使用固定导航：`position: fixed`
- 导航 `z-index` 必须全局最高
- 导航只做轻微状态变化，不参与模糊、缩放、强位移
- 可用 `ScrollTrigger` 根据 section 中线切换 active 态

### 6.3 统一转场语言

离场场景推荐：

- `opacity: 1 -> 0`
- `filter: blur(0px -> 14px)`
- `scale: 1 -> 0.985`
- `yPercent: 0 -> -8`

入场场景推荐：

- `opacity: 0 -> 1`
- `filter: blur(16px -> 0px)`
- `scale: 1.02 -> 1`
- `yPercent: 10 -> 0`

转场覆盖层：

- 使用白雾、浅光、淡雾蒙版
- 单次时长建议 `0.2s - 0.35s`
- 不要长时间白屏，不然像页面卡死

## 7. Text Masking 规范

标题和关键副文案必须按遮罩入场实现，不准整块一起淡入。

推荐：

```ts
SplitText.create(el, {
  type: 'lines',
  mask: 'lines',
  autoSplit: true,
})
```

推荐动画：

- `yPercent: 110 -> 0`
- `opacity: 0 -> 1`
- `stagger: 0.04 - 0.08`

限制：

- 只对标题、关键副标题使用
- 正文段落不要大面积上遮罩，不然信息密度一高就很烦

## 8. 3D 视差规范

### 8.1 场景透视

- 父容器设置 `perspective: 1800px - 2400px`
- 必须在不同层之间拉开速度差，不要全层同速移动

### 8.2 推荐速度系数

| 层级 | 速度系数 |
|------|----------|
| `bg` | `0.45` |
| `mid` | `1.00` |
| `fg` | `1.35` |

### 8.3 推荐效果

- `bg`：低速横移或纵移，偶尔带轻微缩放
- `mid`：作为视觉焦点，承担主体入场和缓慢漂移
- `fg`：承担遮挡、掠过、轻微反向位移、镜头穿越感

## 9. 特殊元素实现规则

### 9.1 玻璃卡片

- 用 `backdrop-filter` 或近似视觉
- 卡片之间要有前后叠放，不要并排像后台管理系统
- 入场优先使用 `blur + y + opacity`

### 9.2 网络图 / 连线图

- 推荐 SVG path
- 线条可通过 `stroke-dashoffset` 或 path draw 进入
- 节点层和线条层应分离，避免一起动得像贴图

### 9.3 高亮胶囊

- 用于关键词强调、CTA 点题
- 保持 `mix-blend-mode: normal`
- 目标是可读，不是炫技发光垃圾

### 9.4 雾层 / 光晕层

- 用于主转场、白场、焦点重建
- 优先作为 `fg` 或单独 overlay 层使用
- 不要全程常驻，不然页面像蒙了磨砂膜

## 10. GSAP 参数预设

```ts
const EASE_ENTER = 'power3.out'
const EASE_EXIT = 'power2.inOut'
const D_ENTER = 0.72
const D_EXIT = 0.52
const STAGGER_TEXT = 0.06

gsap.from(lines, {
  yPercent: 110,
  opacity: 0,
  duration: D_ENTER,
  stagger: STAGGER_TEXT,
  ease: EASE_ENTER,
})
```

如果页面需要更奢一点的惯性，可以考虑：

- `ScrollTrigger`
- `SplitText`
- `CustomEase`
- `Lenis` 或 `ScrollSmoother`

## 11. 实施步骤

每次调用本技能时，按这个顺序做：

1. 盘点 section 数量和每个 section 的主叙事目标
2. 盘点每个 section 的素材是否满足 `bg / mid / fg`
3. 按目录规范整理静态资源
4. 搭建 section 容器和固定导航
5. 先做无动画静态排版，确认响应式和裁切
6. 接入 `ScrollTrigger` 分镜时间轴
7. 实现文字遮罩、分层视差、场景转场
8. 补白场、雾层、前景遮挡等叙事细节
9. 做 `prefers-reduced-motion` 降级
10. 做桌面与移动端验收

## 12. 验收清单

全部满足才算完成：

1. 导航始终可见可点击
2. 每个分镜至少能看出两层以上不同速度位移
3. 标题入场带遮罩，不是整块淡入
4. 主转场存在“白雾过渡 + 焦点重建”
5. `375 / 768 / 1280 / 1440` 下无横向溢出
6. 移动端裁切后主主体仍可辨认
7. `prefers-reduced-motion` 下关闭强位移和强模糊
8. 素材目录、命名、层级符合规范

## 13. 常见失败模式

1. 只做 `opacity`
原因：廉价，像 PPT，不像分镜叙事

2. 所有层同速移动
原因：完全没有空间深度

3. 文案和按钮烘焙进图片
原因：响应式、可读性、交互全废

4. 前景层只是装饰小碎片
原因：没有遮挡关系，做不出穿越感

5. 导航跟着场景一起模糊缩放
原因：用户直接丢失交互锚点

## 14. 快速决策规则

- 需要“高级感”，优先增加层次深度，不优先堆特效
- 需要“可读性”，先减 blur 和位移，再谈复杂转场
- 需要“流畅度”，先降阴影与滤镜成本，再谈更强 3D
- 需要“叙事感”，优先强化 `fg` 遮挡和主焦点切换
