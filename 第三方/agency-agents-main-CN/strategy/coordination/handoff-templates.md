# 📋 NEXUS 交接模板

> NEXUS 管道中每种智能体间交接的标准化模板。一致的交接可防止上下文丢失 — 这是多智能体协调失败的头号原因。

---

## 1. 标准交接模板

用于任何智能体间的工作传递。

```markdown
# NEXUS 交接文档

## 元数据
| 字段 | 值 |
|------|-----|
| **来自** | [智能体名称] ([部门]) |
| **发往** | [智能体名称] ([部门]) |
| **阶段** | 阶段 [N] — [阶段名称] |
| **任务参考** | [来自 Sprint Prioritizer 待办的任务 ID] |
| **优先级** | [关键 / 高 / 中 / 低] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 上下文
**项目**：[项目名称]
**当前状态**：[到目前为止已完成的内容 — 要具体]
**相关文件**：
- [file/path/1] — [包含什么内容]
- [file/path/2] — [包含什么内容]
**依赖项**：[此工作依赖什么完成]
**约束条件**：[技术、时间线或资源约束]

## 交付请求
**需要什么**：[具体、可衡量的交付物描述]
**验收标准**：
- [ ] [标准 1 — 可衡量]
- [ ] [标准 2 — 可衡量]
- [ ] [标准 3 — 可衡量]
**参考材料**：[规格、设计、先前工作的链接]

## 质量期望
**必须通过**：[此交付物的具体质量标准]
**所需证据**：[完成证明是什么样的]
**交接给下一步**：[谁接收输出以及他们需要什么格式]
```

## 2. QA 反馈循环 — 通过

当 Evidence Collector 或其他 QA 智能体批准任务时使用。

```markdown
# NEXUS QA 裁决：通过 ✅

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **尝试** | [第 N 次]，最多 3 次 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：通过

## 证据
**截图**：
- 桌面 (1920x1080)：[文件名/路径]
- 平板 (768x1024)：[文件名/路径]
- 移动 (375x667)：[文件名/路径]

**功能验证**：
- [x] [验收标准 1] — 已验证
- [x] [验收标准 2] — 已验证
- [x] [验收标准 3] — 已验证

**品牌一致性**：已验证 — 颜色、排版、间距匹配设计系统
**无障碍**：已验证 — 键盘导航、对比度、语义化 HTML
**性能**：[测量的加载时间] — 在可接受范围内

## 备注
[任何观察、未来改进的小建议或正面反馈]

## 下一步行动
→ Agents Orchestrator：标记任务完成，推进到待办中的下一个任务
```

## 3. QA 反馈循环 — 失败

当 Evidence Collector 或其他 QA 智能体拒绝任务时使用。

```markdown
# NEXUS QA 裁决：失败 ❌

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **尝试** | [第 N 次]，最多 3 次 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：失败

## 发现的问题

### 问题 1：[类别] — [严重程度：关键/高/中/低]
**描述**：[问题的精确描述]
**预期**：[根据验收标准应该发生什么]
**实际**：[实际发生了什么]
**证据**：[截图文件名或测试输出]
**修复说明**：[具体、可操作的解决说明]
**需修改的文件**：[精确的文件路径]

### 问题 2：[类别] — [严重程度]
**描述**：[...]
**预期**：[...]
**实际**：[...]
**证据**：[...]
**修复说明**：[...]
**需修改的文件**：[...]

[继续列出所有发现的问题]

## 验收标准状态
- [x] [标准 1] — 通过
- [ ] [标准 2] — 失败（见问题 1）
- [ ] [标准 3] — 失败（见问题 2）

## 重试说明
**给开发者智能体**：
1. 仅修复上面列出的问题
2. 不要引入新功能或变更
3. 解决所有问题后重新提交 QA
4. 这是第 [N] 次尝试，最多 3 次

**如果第 3 次尝试仍失败**：任务将升级至 Agents Orchestrator
```

## 4. 升级报告

当任务超过 3 次重试尝试时使用。

```markdown
# NEXUS 升级报告 🚨

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者智能体** | [智能体名称] |
| **QA 智能体** | [智能体名称] |
| **已用尽尝试** | 3/3 |
| **升级至** | [Agents Orchestrator / Studio Producer] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 失败历史

### 第 1 次尝试
- **发现的问题**：[摘要]
- **应用的修复**：[开发者做了什么更改]
- **结果**：失败 — [为什么仍然失败]

### 第 2 次尝试
- **发现的问题**：[摘要]
- **应用的修复**：[开发者做了什么更改]
- **结果**：失败 — [为什么仍然失败]

### 第 3 次尝试
- **发现的问题**：[摘要]
- **应用的修复**：[开发者做了什么更改]
- **结果**：失败 — [为什么仍然失败]

## 根因分析
**为什么任务持续失败**：[潜在问题的分析]
**系统性问题**：[这是偶发还是模式？]
**复杂度评估**：[任务范围是否合理？]

## 建议解决方案
- [ ] **重新分配** 给不同的开发者智能体（[推荐的智能体]）
- [ ] **分解** 为更小的子任务（[建议的拆分方式]）
- [ ] **修订方法** — 需要架构/设计变更
- [ ] **接受** 当前状态并记录已知限制
- [ ] **推迟** 到未来冲刺

## 影响评估
**阻塞**：[哪些其他任务被此阻塞]
**时间线影响**：[这如何影响整体进度]
**质量影响**：[如果接受当前状态存在什么质量妥协]

## 需要的决策
**决策者**：[Agents Orchestrator / Studio Producer]
**截止日期**：[何时需要决策以避免进一步延误]
```

## 5. 阶段门交接

在 NEXUS 阶段之间转换时使用。

```markdown
# NEXUS 阶段门交接

## 转换
| 字段 | 值 |
|------|-----|
| **从阶段** | 阶段 [N] — [名称] |
| **到阶段** | 阶段 [N+1] — [名称] |
| **门守** | [智能体名称] |
| **门结果** | [通过 / 失败] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 门标准结果
| # | 标准 | 阈值 | 结果 | 证据 |
|---|------|------|------|------|
| 1 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 2 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |
| 3 | [标准] | [阈值] | ✅ 通过 / ❌ 失败 | [证据参考] |

## 向前传递的文档
1. [文档名称] — [下一阶段的用途]
2. [文档名称] — [下一阶段的用途]
3. [文档名称] — [下一阶段的用途]

## 下一阶段的关键约束
- [来自本阶段发现的约束 1]
- [来自本阶段发现的约束 2]

## 下一阶段的智能体激活
| 智能体 | 角色 | 优先级 |
|--------|------|--------|
| [智能体 1] | [下一阶段中的角色] | [立即 / 第 2 天 / 按需] |
| [智能体 2] | [下一阶段中的角色] | [立即 / 第 2 天 / 按需] |

## 向前传递的风险
| 风险 | 严重程度 | 缓解措施 | 负责人 |
|------|----------|----------|--------|
| [风险] | [P0-P3] | [缓解计划] | [智能体] |
```

| **速度** | [计划] / [实际] 故事点 |

## 完成状态
| 任务 ID | 描述 | 状态 | QA 尝试 | 备注 |
|---------|------|------|---------|------|
| [ID] | [描述] | ✅ 完成 | [N] | [备注] |
| [ID] | [描述] | ✅ 完成 | [N] | [备注] |

## 结转至下一冲刺

## 7. 事件交接

在事件响应期间使用。

```markdown
# NEXUS 事件交接

## 事件
| 字段 | 值 |
|------|-----|
| **严重程度** | [P0 / P1 / P2 / P3] |
| **发现者** | [智能体或系统] |
| **发现时间** | [时间戳] |
| **分配给** | [智能体名称] |
| **状态** | [调查中 / 缓解中 / 已解决 / 事后分析] |

## 描述
**发生了什么**：[事件的清晰描述]
**影响**：[谁/什么受到影响以及严重程度]
**时间线**：
- [HH:MM] — [事件]
- [HH:MM] — [事件]
- [HH:MM] — [事件]

## 当前状态
**受影响系统**：[列表]
**是否有临时方案**：[是/否 — 如果是请描述]
**预计解决时间**：[时间估计]

## 已采取的行动
1. [已采取的行动及结果]
2. [已采取的行动及结果]

## 交接上下文
**给下一响应者**：
- [已尝试了什么]
- [尚未尝试什么]
- [疑似根因]
- [需要检查的相关日志/指标]

## 利益相关者沟通
**上次更新发送**：[时间戳]
**下次更新到期**：[时间戳]
**沟通渠道**：[更新发布在哪里]
```

## 使用指南

| 场景 | 使用的模板 |
|------|-----------|
| 向另一个智能体分配工作 | 标准交接 (#1) |
| QA 批准任务 | QA 通过 (#2) |
| QA 拒绝任务 | QA 失败 (#3) |
| 任务超过 3 次重试 | 升级报告 (#4) |
| 阶段之间移动 | 阶段门交接 (#5) |
| 冲刺结束 | 冲刺交接 (#6) |
| 系统事件 | 事件交接 (#7) |
