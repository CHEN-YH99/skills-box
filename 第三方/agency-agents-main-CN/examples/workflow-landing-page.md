# 多智能体工作流：落地页冲刺

> 使用4个智能体在一天内交付一个转化优化的落地页。

## 场景

你需要为新产品发布制作一个落地页。它需要外观出色、转化访客，并在当天上线。

## 智能体团队

| 智能体 | 在此工作流中的角色 |
|-------|---------------------|
| 内容创作者 | 撰写文案 |
| UI设计师 | 设计布局和组件规范 |
| 前端开发者 | 构建页面 |
| 增长黑客 | 优化转化 |

## 工作流

### 上午：文案 + 设计（并行）

**步骤1a — 激活内容创作者**

```
激活内容创作者。

为"FlowSync"撰写落地页文案——一个API集成平台，
可在5分钟内连接任意两个SaaS工具。

目标受众：中型公司的开发者和技PM。
语调：自信、简洁、略带俏皮。

需要的部分：
1. 英雄区（标题 + 副标题 + CTA）
2. 问题陈述（3个痛点）
3. 工作原理（3个步骤）
4. 社会证明（占位推荐格式）
5. 定价（3个层级：免费、专业、企业）
6. 最终CTA

保持可扫描。不要废话。
```

**步骤1b — 激活UI设计师（并行）**

```
激活UI设计师。

为SaaS落地页设计规范。产品：FlowSync（API集成平台）。
风格：干净、现代、暗色模式选项。参考Linear或Vercel的美学。

交付：
1. 布局线框图（区块顺序 + 间距）
2. 色彩方案（主色、辅色、强调色、背景色）
3. 排版（字体搭配、标题大小、正文大小）
4. 组件规范：英雄区、功能卡片、定价表、CTA按钮
5. 响应式断点（移动端、平板、桌面）
```

### 中午：构建

**步骤2 — 激活前端开发者**

```
激活前端开发者。

根据以下规范构建落地页：

文案：[粘贴内容创作者输出]
设计：[粘贴UI设计师输出]

技术栈：HTML、Tailwind CSS、最少的原生JS（无需框架）。
要求：
- 响应式（移动端优先）
- 快速（无重型资源，系统字体即可）
- 无障碍（正确的标题层级、alt文本、焦点状态）
- 包含可用的邮箱注册表单（action URL: /api/subscribe）

交付一个可直接部署的index.html文件。
```

### 下午：优化

**步骤3 — 激活增长黑客**

```
激活增长黑客。

审阅此落地页的转化优化：

[粘贴HTML或描述当前页面]

评估：
1. CTA是否在首屏之上？
2. 价值主张是否在5秒内清晰？
3. 注册流程是否有摩擦？
4. 你会优先运行哪些A/B测试？
5. SEO基础：meta标签、OG标签、结构化数据

给我具体的修改建议，不是泛泛的建议。
```

## 时间线

| 时间 | 活动 | 智能体 |
|------|------|--------|
| 9:00 | 文案 + 设计启动（并行） | 内容创作者 + UI设计师 |
| 11:00 | 构建开始 | 前端开发者 |
| 14:00 | 第一版就绪 | — |
| 14:30 | 转化审阅 | 增长黑客 |
| 15:30 | 应用反馈 | 前端开发者 |
| 16:30 | 上线 | 部署到Vercel/Netlify |

## 关键模式

1. **并行启动**：文案和设计同时进行，因为它们相互独立
2. **合并点**：前端开发者需要两个输出后才能开始
3. **反馈循环**：增长黑客审阅，然后前端开发者应用修改
4. **时间盒**：每个步骤有明确的时间盒，防止范围蔓延
