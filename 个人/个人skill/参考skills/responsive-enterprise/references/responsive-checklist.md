# Responsive Checklist

在完成页面或组件的响应式改造后，按这份清单做交付前检查。

## 1. 视口覆盖

- 手机小屏：`320-375px`
- 手机大屏：`390-430px`
- 平板竖屏：`768px`
- 平板横屏 / 小笔记本：`1024px`
- 常规桌面：`1280-1440px`
- 大屏桌面：`1600px+`（如果页面是营销页、看板、数据后台则必须看）

## 2. 全局硬伤

- 页面没有非预期横向滚动条
- 主内容没有被固定头部、底部、侧边栏遮挡
- 文本、按钮、输入框、徽标、图片没有裁切
- 页面缩放到 `200%` 时仍可用
- 键盘焦点可见且不会跳到屏幕外

## 3. 布局稳定性

- 栅格列数能自然减少，不靠隐藏主要内容维持体面
- `flex` 容器允许换行，换行后顺序和对齐仍合理
- 卡片高度变化时不会把操作按钮挤飞
- 主次栏位在窄屏下堆叠逻辑清晰
- 粘性元素不会把可交互区域压没

## 4. 文案与可读性

- 标题不会在小屏把首屏占满
- 正文行长与字号在移动端仍可读
- 中英文混排、长链接、长按钮文案不会撑爆容器
- 数字、价格、标签、状态文案在多语言长度变化下仍稳定

## 5. 组件专项

### 导航

- Logo、导航项、CTA 不会互相顶爆
- 移动端收起后，关键入口仍可直达
- 展开菜单可滚动、可关闭、焦点不丢

### 表单

- 行内表单在窄屏改为纵向排列
- 错误提示不会把布局冲烂
- 输入控件高度与点击区域足够

### 表格 / 数据区

- 优先考虑信息重排，而不是直接整块横向滚动
- 如果必须横向滚动，表头、操作列、分页仍可用
- 关键字段在小屏不会消失得莫名其妙

### 弹层 / 抽屉

- 弹层高度不会超出可视区
- 内容区能滚动，头尾操作区可达
- 打开和关闭后页面滚动状态合理

## 6. 媒体资源

- 图片默认 `max-width: 100%`
- 背景图和插画不会制造横向溢出
- `object-fit` 不会裁掉关键信息
- 视频、轮播、canvas 在移动端仍保持可见和可操作

## 7. 代码质量

- 没有到处散落的魔法断点值
- 同类问题优先从父级布局修复，不是对子元素贴满补丁
- 优先复用现有 token、spacing、breakpoint 变量
- 没有用 `overflow: hidden` 掩盖真实 bug

## 8. 交付说明

输出结果时至少说明：

- 修了哪些页面或组件
- 采用了哪些断点或流式策略
- 做了哪些视口验证
- 还有哪些剩余风险没法在当前环境完全验证
