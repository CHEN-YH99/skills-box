# 工作流示例：书籍章节开发

> 一个聚焦的单智能体工作流，用于将粗略的原始素材转化为带有显式修订循环的战略性第一人称章节草稿。

## 何时使用此工作流

当作者有语音笔记、片段或战略性笔记，但还没有干净的章节草稿时使用此工作流。目标不是通用的代笔写作。目标是产出一个强化品类定位、保留作者声音、并清晰暴露未决编辑决策的章节。

## 使用的智能体

| 智能体 | 角色 |
|-------|------|
| 书籍合著者 | 将原始素材转化为带编辑笔记和后续步骤问题的版本化章节草稿 |

## 示例激活

```text
激活书籍合著者。

书籍目标：围绕面向中等企业（Mittelstand）的实用AI采纳建立权威。
目标读者：20-200人企业的所有者和运营领导者。
章节主题：为什么大多数AI项目在实施开始之前就失败了。
期望的草稿成熟度：首个实质性草稿。

原始素材：
- 语音备忘录："真正的失败发生在期望设定，而不是工具选择。"
- 笔记：领导者在定义运营瓶颈之前就购买了软件。
- 故事片段：我们差点在一个橱柜制作工作流中推出了错误的自动化，因为实际问题出在报价延迟，而不是生产吞吐量。
- 定位角度：实用现实主义胜过炒作。

请产出：
1. 章节目标及其在书中的战略角色
2. 你需要澄清的问题
3. 第2章 - 版本1 - 准备审阅
4. 关于假设和证据空白的编辑笔记
5. 具体的后续修订请求
```

## 预期输出结构

书籍合著者应以五个部分回应：

1. `目标成果`
2. `章节草稿`
3. `编辑笔记`
4. `反馈循环`
5. `下一步`

## 质量标准

- 草稿保持第一人称声音
- 章节有一个清晰的承诺和内在逻辑
- 论点与原始素材关联或标记为假设
- 移除通用的激励性语言
- 输出以明确的修订问题结束，而非模糊的交接
