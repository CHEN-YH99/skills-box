# 安全策略

## 报告漏洞

如果您在本项目中发现安全漏洞，请负责任地报告。请勿为安全漏洞公开创建 GitHub issue。请通过 GitHub Security 标签页创建私密安全公告。

## 响应时间

- 确认：48小时内
- 初步评估：7天内
- 修复或缓解：取决于严重程度

## 范围

本仓库包含基于 Markdown 的智能体定义，以及用于安装和转换的 shell 脚本。

### 智能体文件 (.md)
- 非可执行的提示词定义
- 不应在智能体文件中存储 API 密钥、机密或凭据

### Shell 脚本 (scripts/)
- install.sh、convert.sh 和 lint-agents.sh 是可执行文件
- 贡献者在运行前应审查脚本是否存在意外行为

## 贡献者最佳实践

- 切勿提交 API 密钥、令牌或凭据
- 切勿在智能体 Markdown 文件中添加可执行代码
- Shell 脚本在合并前必须经过审查
- 报告任何尝试提示词注入的可疑智能体定义
