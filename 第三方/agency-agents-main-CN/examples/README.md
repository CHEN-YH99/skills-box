# 示例

本目录包含示例输出，展示如何将机构中的智能体协调在一起处理真实世界任务。

## 为什么需要这个目录

agency-agents 仓库定义了横跨工程、设计、营销、产品、支持、空间计算和项目管理等领域的数十个专业智能体。但仅凭智能体定义并不能展示当你**同时部署它们**执行同一任务时会发生什么。

这些示例回答了这个问题：*"当整个机构协作时，实际看起来是什么样的？"*

## 内容

### [nexus-spatial-discovery.md](./nexus-spatial-discovery.md)

**内容：** 一次完整的产品发现练习，8个智能体并行工作，评估一个软件机会并产出统一计划。

**场景：** 网络研究发现了AI智能体编排与空间计算交汇处的一个机会。随后整个机构被同时部署以产出：

- 市场验证和竞争分析
- 技术架构（8服务系统设计，含完整SQL模式）
- 品牌策略和视觉识别
- 上市和增长计划
- 客户支持运营蓝图
- UX研究计划，含用户画像和旅程地图
- 35周项目执行计划，含65个冲刺工单
- 空间界面架构规范

**使用的智能体：**
| 智能体 | 角色 |
|-------|------|
| 产品趋势研究员 | 市场验证、竞争格局 |
| 后端架构师 | 系统架构、数据模型、API设计 |
| 品牌守护者 | 定位、视觉识别、命名 |
| 增长黑客 | GTM策略、定价、发布计划 |
| 支持响应者 | 支持层级、入门引导、社区 |
| UX研究员 | 用户画像、旅程地图、设计原则 |
| 项目管理者 | 阶段计划、冲刺、风险登记 |
| XR界面架构师 | 空间UI规范 |

**核心要点：** 所有8个智能体并行运行，产出了连贯的、交叉引用的计划，无需协调开销。输出展示了机构在单次会话中从"发现机会"到"完整蓝图"的能力。

## 添加新示例

如果你运行了一个有趣的多智能体练习，考虑将其添加到这里。好的示例应展示：

- 多个智能体在共同目标上协作
- 机构能力的广度
- 智能体定义在真实场景中的适用性
